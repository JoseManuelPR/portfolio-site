## La restricción

El trabajo de campo agrícola ocurre en zonas remotas donde la conectividad a internet es poco confiable o inexistente. AGROS necesitaba recolectar datos, capturar coordenadas GPS y verificar la identidad de los trabajadores — todo sin conexión de red.

## Decisiones de arquitectura

Elegimos una arquitectura sync-first donde el dispositivo es la fuente de verdad hasta que pueda alcanzar el servidor:

```typescript
// Cola de sincronización simplificada
interface SyncQueue {
  pending: DataRecord[];
  synced: string[]; // IDs de registros sincronizados exitosamente
  conflicts: ConflictRecord[];
}

async function syncWhenOnline(queue: SyncQueue) {
  const connection = await checkConnectivity();
  if (!connection) return;

  for (const record of queue.pending) {
    try {
      await api.upload(record);
      queue.synced.push(record.id);
    } catch (e) {
      if (isConflict(e)) {
        queue.conflicts.push({ record, serverVersion: e.data });
      }
    }
  }
}
```

## Georreferenciación sin señal GPS

En vegetación densa, la precisión del GPS cae significativamente. Implementamos una estrategia de ubicación multi-fuente:

1. GPS (cuando está disponible y es preciso)
2. Triangulación de torres celulares (aproximada pero confiable)
3. Última posición conocida con decaimiento temporal
4. Pin manual sobre tiles de mapa cacheados

## Identificación biométrica

La identificación de trabajadores en campo necesitaba funcionar offline. Usamos matching de huellas on-device con plantillas almacenadas localmente, sincronizando los registros de verificación cuando volvía la conectividad.

## Resolución de conflictos

La parte más difícil del offline-first es manejar conflictos cuando dos dispositivos modifican los mismos datos. Nuestra estrategia: last-write-wins para campos simples, merge para colecciones y resolución manual para datos críticos.

## Conclusiones

Construir para offline no es solo cachear — es repensar cada interacción asumiendo que la red no existe. El resultado fue una app en la que los trabajadores de campo confiaban, porque nunca perdía sus datos.
